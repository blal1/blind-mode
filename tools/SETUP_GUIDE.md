# YGO Master Duel Accessibility Mod — Setup Guide

## Required Tools

### 1. BepInEx 6 IL2CPP (Mod Framework)
- Purpose: Loads mods into IL2CPP Unity games; generates managed interop stubs from the native binary
- Download: pre-configured package is in `INSTALL/` at the repo root — copy it to the game folder
- Manual download: https://builds.bepinex.dev/projects/bepinex_be (BepInEx IL2CPP bleeding-edge)
- Installation: extract into the game folder so `BepInEx/` sits next to `Master Duel.exe`

### 2. ilspycmd (Command-line Decompiler)
- Purpose: Decompile specific types from BepInEx interop stubs (managed .NET DLLs)
- Install: `dotnet tool install -g ilspycmd`
- Usage: `.\tools\decompile.ps1 "YgomGame.Duel.DuelLP"` (wrapper script handles paths)
- Note: do NOT open `GameAssembly.dll` — it is a native IL2CPP binary. Use the stubs in `BepInEx\interop\` instead.

### 3. ILSpy / dnSpy (GUI Decompiler — optional)
- Purpose: Browse interop stubs visually
- ILSpy download: https://github.com/icsharpcode/ILSpy/releases
- dnSpy download: https://github.com/dnSpy/dnSpy/releases
- Load files from: `BepInEx\interop\Assembly-CSharp.dll`

### 4. Il2CppDumper (Type Dump — already done)
- Purpose: Generates `dump\dump.cs` — authoritative text reference for all game types and method signatures
- The dump is already committed at `dump\dump.cs` (~35 MB)
- Refresh after game updates: re-run Il2CppDumper against `GameAssembly.dll`
- Download: https://github.com/Perfare/Il2CppDumper

### 5. Visual Studio 2022 or VS Code + C# Dev Kit
- Purpose: Write and build mod code
- Required workload: .NET desktop development (VS) or C# Dev Kit extension (VS Code)
- Target framework: .NET 6.0 (matches the mod project)

### 6. Tolk / TolkDotNet (Screen Reader Bridge)
- Purpose: Speaks text via NVDA, JAWS, or SAPI fallback
- Pre-bundled at: `BepInEx\plugins\dependencies\TolkDotNet.dll`
- Native libs at: `BepInEx\plugins\dependencies\lib\screen-reader-libs\windows\`
- Source: https://github.com/ndarilek/tolk

## Installation Steps

### Step 1: Install BepInEx
1. Copy the contents of `INSTALL\` to the game folder:
   `C:\Program Files (x86)\Steam\steamapps\common\Yu-Gi-Oh!  Master Duel\`
   so that `BepInEx\` sits next to `Master Duel.exe`
2. Run the game once — BepInEx will generate `BepInEx\interop\` stubs on first launch

### Step 2: Verify BepInEx Installation
After first run, check for:
- `BepInEx\interop\Assembly-CSharp.dll` (the managed stubs)
- `BepInEx\LogOutput.log` (BepInEx console output)
- `BepInEx\plugins\` folder (where the mod DLL goes)

### Step 3: Set Up Development Environment
1. Clone this repo
2. Open `mod\MasterDuelAccessibility\MasterDuelAccessibility.csproj` in VS / VS Code
3. Run `dotnet restore` to fetch NuGet packages
4. Run `dotnet build` — on Windows this auto-copies the DLL to `BepInEx\plugins\`

### Step 4: Analyze Game Code
1. Search `dump\dump.cs` for type/method names before writing any patch:
   ```
   Select-String -Path "dump\dump.cs" -Pattern "DuelLP"
   ```
2. Decompile a specific type for full source:
   ```
   .\tools\decompile.ps1 "YgomGame.Duel.DuelLP"
   ```
3. Run the analysis tools for broader discovery:
   ```
   powershell -NoProfile -File tools\analyze_core_menus.ps1
   powershell -NoProfile -File tools\analyze_input.ps1
   ```
4. Use `AssemblyAnalyzer` for a categorised HTML-style report:
   ```
   tools\AssemblyAnalyzer\AssemblyAnalyzer.exe
   ```

## Next Steps After Setup

1. Search `dump\dump.cs` for the class you want to patch
2. Decompile it with `.\tools\decompile.ps1 "Full.Type.Name"`
3. Write a Harmony postfix in `mod\MasterDuelAccessibility\Patches\`
4. Register the patch in `Plugin.ApplyPatches()` (static patches) or `LatePatches.ApplyDuelScenePatches()` (duel-only patches)
5. Call `tts.Speak(Loc.Get("your_key"), interrupt: true)` to announce via screen reader
6. Add localization strings for `"your_key"` in `Loc.cs`
7. Build with `dotnet build` and launch the game to test
