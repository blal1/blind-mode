# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is an **accessibility mod development workspace** for Yu-Gi-Oh! Master Duel, built by a blind developer using AI assistance. The repository has two main components:

1. **`mod/MasterDuelAccessibility/`** — The actual BepInEx 6 IL2CPP mod (C#, .NET 6, x64)
2. **`Accessibility-Mod-Template/`** — An AI-friendly template/framework for creating accessibility mods (fork of HappyStarfish/Accessibility-mod-template, normalized for Claude Code)

## Build Commands

```bash
# Build the mod
cd "mod/MasterDuelAccessibility"
dotnet build

# Restore NuGet packages (needed first time or after csproj changes)
dotnet restore

# Build in Release mode
dotnet build -c Release
```

The build auto-copies the DLL to the game's BepInEx plugins folder **on Windows only**. On Linux/WSL the build succeeds but the copy step is skipped.

**Output:** `mod/MasterDuelAccessibility/bin/Debug/net6.0/MasterDuelAccessibility.dll`

## Mod Architecture

The mod uses **Harmony postfix patches** applied via runtime reflection (`AccessTools.TypeByName`) — no compile-time game DLL dependencies except the interop stubs.

### Key files

| File | Role |
|------|------|
| `Plugin.cs` | BepInEx entry point — initializes Tolk, registers `KeyboardShortcuts` MonoBehaviour, applies all Harmony patches |
| `TtsService.cs` | Tolk wrapper (NVDA/JAWS/SAPI). `Speak(text, interrupt)`, `Repeat()`, `Silence()`. Calls `SetDllDirectory` to point at `lib/screen-reader-libs/windows/` |
| `KeyboardShortcuts.cs` | IL2CPP-injected `MonoBehaviour` — handles F1–F5, Space, Alt in `Update()`. Tracks `IsInDuel`, `MyLP`, `OppLP` |
| `AccessToolsHelper.cs` | `FindType(name)` — safe wrapper around `AccessTools.TypeByName` |
| `NullablePolyfill.cs` | Required shim so `?` nullable annotations compile against IL2CPP interop DLLs |
| `Patches/DuelHudPatch.cs` | `DuelLP.ChangeLP` / `SetLP` — announces LP changes, updates `KeyboardShortcuts` state |
| `Patches/DuelEventPatch.cs` | `DuelClient.RunEffect` (viewType switch), `Awake`/`OnDestroy` for duel start/end |
| `Patches/PhasePatch.cs` | `PhaseSelectWindow.PhaseChange` / `TurnChange` |
| `Patches/CardInfoPatch.cs` | `CardInfo.SetDescriptionArea` — reads `m_CardName_Property` via reflection |

### Keyboard shortcuts

| Key | Context | Action |
|-----|---------|--------|
| Space (32) | Duel | Read LP for both players |
| Alt (308) | Anywhere | Read current card info |
| F1 (282) | Anywhere | Read shortcut help |
| F2 (283) | Anywhere | Repeat last spoken text |
| F3 (284) | Duel | Read hand cards |
| F4 (285) | Duel | Read field cards |
| F5 (286) | Anywhere | Silence/stop speech |

### Patching approach

Patches are applied in `Plugin.ApplyPatches()` using fully-qualified type names (e.g. `"YgomGame.Duel.DuelLP"`). Postfix methods receive `object __instance` — use reflection to access fields. This avoids a compile-time dependency on `Assembly-CSharp.dll`.

## Discovering Game APIs

**Before implementing any new patch:**

1. Search `dump/dump.cs` for class/method names — this is the authoritative source for exact signatures
2. Search `decompiled/interop/` for generated stubs (use `ilspycmd -t "TypeName"` for targeted decompilation)
3. Use `ilspycmd` to decompile specific types: `ilspycmd "BepInEx/interop/Assembly-CSharp.dll" -t "YgomGame.Duel.TypeName"`
4. Never guess method names — always verify against the dump

```bash
# Decompile a specific type
ilspycmd "BepInEx/interop/Assembly-CSharp.dll" -t "YgomGame.Duel.ClassName"

# List all classes in a namespace
ilspycmd "BepInEx/interop/Assembly-CSharp.dll" -l class | grep "YgomGame.Duel"

# Search dump for a keyword
grep -i "keyword" dump/dump.cs | head -30
```

## Tolk / TTS Setup

Tolk.dll and its dependencies (`nvdaControllerClient64.dll`, `SAAPI64.dll`) are at:
`BepInEx/plugins/dependencies/lib/screen-reader-libs/windows/`

`TtsService` calls `SetDllDirectory` at init so the OS finds these DLLs. When deploying to the game, this folder must exist relative to `MasterDuelAccessibility.dll`.

`TolkDotNet.dll` is at `BepInEx/plugins/dependencies/TolkDotNet.dll`.
